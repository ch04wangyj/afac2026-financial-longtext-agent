# LogicRAG current-vs-paper gap matrix

Source of paper contract: `theory/references/notes/2026-06-19_logicrag-2508.06105v2-grounded-spec.md`
Paper: arXiv:2508.06105v2
Audit scope: current implementation files requested in task body.

## Executive readout

Current AFAC2026 LogicRAG already has three paper-aligned skeleton pieces: runtime LLM decomposition, DAG acyclicity/topological scheduling, and same-rank merged retrieval. The main paper-faithfulness gaps are elsewhere: the retrieval backend is sparse BM25/RRF rather than embedding-based, execution never performs dynamic DAG augmentation when context is insufficient, and the solver composes from accumulated raw evidence plus rank summaries instead of using the paper's node-resolution + rolling-memory-forward contract.

Recommended Phase 1 boundary:
- Keep: LLM planning, DAG sanitation/topological levels, same-rank merge scheduling, rolling-memory summarization scaffolding.
- Refactor for paper faithfulness: retrieval backend contract, rank execution semantics, dynamic augmentation path, anti-hesitation/progression controls as explicit policy, final compose inputs.
- Postpone to Phase 2 or gate behind adaptation flags: sparse lexical target engineering, feedback query expansion, context pack heuristics, A-board option-specific retry logic.

## Cross-file conclusions

### Faithful already enough to preserve in Phase 1
1. LLM-based runtime decomposition exists.
2. DAG dependency validation/topological scheduling exists.
3. Same-rank merge/unified retrieval scaffolding exists.
4. Rolling memory summarization scaffolding exists.

### Core paper gaps that need Phase 1 refactor
1. Retrieval backend is sparse BM25/RRF, not embedding-backed.
2. No dynamic DAG augmentation on insufficient context.
3. Solver does not resolve node/subproblem answers in the paper's forward-pass shape.
4. Final compose still consumes accumulated raw evidence, weakening the paper's summary-based context-pruning contract.
5. Anti-hesitation / sampling-without-replacement is only implicit via loop structure, not an explicit execution policy.

### Adaptations that should be gated or moved to Phase 2
1. Lexical target engineering (`targets.py`).
2. Seed-result lexical feedback expansion (`expansion.py`).
3. Deterministic context packs (`context.py`).
4. Heuristic sparse reranking (`rerank.py`).
5. A-board multi-option retry and coverage logic in `solver.py` / runtime config.

## Suggested implementation order

1. Refactor `agent/retrieve/retriever.py` around a retrieval backend interface that can express an embedding retriever as the paper-default path.
2. Refactor `agent/reasoning/logicrag.py` to represent unresolved-node detection and dynamic node append/update explicitly.
3. Refactor `agent/reasoning/solver.py` so rank execution produces resolved node outputs and uses rolling summarized memory as the primary carry-forward state.
4. Separate paper-core runtime settings from AFAC adaptation flags in `agent/runtime/logicrag_config.py`, `configs/logicrag_runtime.yaml`, and `agent/config.py`.
5. Reintroduce sparse-target/rerank/context helpers only as optional adaptation modules after the paper-faithful path is locked.
