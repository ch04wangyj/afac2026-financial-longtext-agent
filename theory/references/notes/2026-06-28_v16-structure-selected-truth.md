# V16 结构导航与逐项真值组装

## 1. 目标与评分门槛

V14 官网得分为 `68.6873`，本地提交 Token 为 `312,541`。代入官方公式可唯一反推 V14 为 `70/100` 正确。V16 当前 Token 为 `315,727`，若达到 `82/100`，预计综合分约为 `80.45`。因此本轮目标是相对 V14 净增至少 12 题，而不是继续压缩少量 Token。

## 2. V15 失败复盘

- 全题 PoT、thinking、自验证和 LLM Judge 将 Token 提高到 `637,718`，并产生 52 题答案漂移。
- 旧视觉表格候选只人工覆盖 5 题，官网从 V14 的 70 题反推下降到约 68 题。
- 根因不是单一 BM25 召回率，而是文档身份缺失、选项跨产品串证据、反事实选项牵引查询，以及最终 Judge 在压缩后丢失选项级证据。

结论：全局重判和单次“更强推理”不再进入主线。所有新模块必须是可消融的增量旁路，答案变化必须有直接证据闭环。

## 3. 近期方法横向比较

| 方法 | 可借鉴部分 | 本赛题不直接采用的部分 | 决策 |
|---|---|---|---|
| [PageIndex](https://github.com/VectifyAI/PageIndex) | 无向量层级树检索、先导航后取证 | 依赖 LLM 构建完整目录树会增加离线复杂度 | 实现 PageIndex-lite 页面/章节导航 |
| [LongRefiner](https://arxiv.org/abs/2505.10413) | 分层文档精炼、先粗筛再细化 | 训练式或额外模型精炼不符合 Qwen-only/成本目标 | 采用确定性节点筛选与邻页展开 |
| [BookRAG](https://arxiv.org/abs/2512.03413) | 结构树与实体关系帮助超长文档定位 | 通用实体图构建成本高，A 榜已有 `doc_ids` | 只保留文档实体别名和局部结构关系 |
| [Structural Deep Encoding for Table QA](https://aclanthology.org/2025.findings-acl.121/) | 表格层级、行列关系比纯文本序列更重要 | 神经编码器与训练不允许 | 延续 V14 表头/单位/跨页行绑定 |
| [RITT](https://aclanthology.org/2025.trl-1.8/) | 迭代式表格检索与证据重排 | 全题多轮模型检索成本和漂移风险高 | 仅对不确定选项预留迭代接口 |
| [Partial-Table Question Generation](https://aclanthology.org/2025.trl-1.19/) | 用局部表格语义增强表格召回 | 生成式训练数据不适合当前冲榜周期 | 将表名、表头、单位、行名写入索引字段 |
| [Document VQA RAG](https://arxiv.org/abs/2508.18984) | 检索与 rerank 往往比单独版面切块更决定效果 | 视觉端到端方案超出当前约束 | V14 版面块必须与检索/文档绑定联动 |

GraphRAG 不进入主线：A 榜已提供候选 `doc_ids`，多数问题是局部条款、数值和跨文档比较，构建全局知识图的收益低于实体绑定成本。LogicRAG 保留实验代码，但其 DAG/多轮 Judge 在本数据上未超过原子谓词检索。

## 4. V16 实现

### 4.1 PageIndex-lite

`agent/retrieve/structure_navigator.py` 按文档和页面构建导航节点，索引标题、章节、条款号、表名、表头、额外字段和正文。查询先选节点，再展开命中页与相邻页。导航结果只加入候选集合，不删除全局 BM25F 结果，避免结构判断错误导致召回损失。

### 4.2 文档实体绑定

`agent/data/document_aliases.py` 将保险 `1..16`、财报公司和研报 pack 映射为业务名称。证据提示显示业务标题，不再只显示 `doc=1`。选项明确点名“太保”“平安e生保”或 `fc_text_003` 时，只在对应文档验证该选项。

### 4.3 逐选项真值组装

`PreciseVerifier` 要求 Qwen 判断“该选项在当前题干选择语义下是否应被选中”，而不是判断括号内解释是否孤立成立。多选题只组装 `true` 选项，`uncertain` 不自动视为正确；模型输出非法或无真选项时回退原答案。

### 4.4 保守融合

候选运行一度产生 54 题变化。最终 `configs/v16_structure_reviews.json` 只接受 14 题，覆盖保险免赔额/施救费用、合同发行与违约条款、监管期限和研报指标；每条都有原文页码或条款理由。其余变化全部回退 V14。
