# V12 文档级高召回验证器与反证审计

> 状态：代码、100 题实跑和提交 CSV 已完成，待官网验证。

## 1. 为什么需要重构

V11 官方得分为 `58.1679`，提交 Token 为 `1,030,141`。按官方公式反推，准确率为 `62/100`。榜首约 98 分，说明继续微调 Top-K 或 RRF 权重不足以弥合差距。

V11 的结构性问题是：相似度检索先裁掉大量上下文，后续 verifier 无法判断从未看到的证据。高置信度也不能表示证据完整。

## 2. 研究依据

- [Chain-of-Agents](https://arxiv.org/html/2406.02818) 指出 RAG 的输入缩减无法保证覆盖所需信息，并通过 worker 依次处理分段、manager 汇总证据，在长上下文任务上优于 RAG 和 full-context 基线。
- [ToM: Tree-oriented MapReduce](https://arxiv.org/html/2511.00489v1) 指出平面 RAG 容易破坏跨片段逻辑一致性，提出按文档层级执行 bottom-up rationale aggregation 和冲突消解。
- [Alibaba Cloud Model Studio 模型文档](https://www.alibabacloud.com/help/en/model-studio/models) 列出 Qwen3.6-Plus 的 1M context 能力；Qwen3.7-Max 官方搜索结果同样标明 1M context。V12 单题证据预算约 5.2 万字符，远低于模型窗口。

ToM 原实现包含 embedding 和额外小模型，不符合本赛题规则。V12 只借鉴“层级归并与冲突消解”，底层继续使用确定性结构和 Qwen API。

## 3. V12 架构

1. 为每个选项构建 claim target。
2. 对每个 query bundle 和每个 `doc_id` 独立执行 BM25F，保证跨文档比较端点均有候选。
3. 在指定文档内扫描非年份数值、完整日期和强实体；普通年份不得单独触发 exact sweep。
4. 展开高分 anchor 的同页及相邻 chunk，恢复表头、例外和上下文。
5. 按选项轮询合并证据，避免 A/B 先耗尽全局字符预算。
6. Qwen3.7-Max 先判断选项事实真伪，再解释题干选择规则并生成严格答案。
7. 对与 V11 不一致的题执行独立反证审计。

## 5. 运行中发现并修复的问题

### 港股股息术语漏召回

`fin_a_013` 中比亚迪 2025 年年报第 2 页披露“建议派发二零二五年末期股息每10股人民币3.58元”。宽泛的分红别名查询被“每股收益3.58元”等财务行干扰。

修复：增加高权重 `dividend_final` 查询，覆盖“末期股息、建议派发、每10股、每股现金分红”。

### 末期方案与全年合计混淆

`fin_a_016` 中美的集团年末拟派 38 元，但包含中期分红的全年方案为 43 元。模型曾用全年合计否定末期方案。

修复：在裁决 contract 中强制区分末期/年末拟派方案与全年合计。

### 年份污染 exact sweep

普通年份在年报中出现过密，会把目录和无关页错误放大。

修复：只有非年份数值、完整日期或至少两个强实体共同命中时才触发 exact sweep；补扫结果只作为候选，不再压过多查询 RRF。

## 6. 已知边界与下一步

V12 将准确率从 62% 提升到 67%，证明扩大文档内证据覆盖有效；但 Token 从 103 万增至 229 万，使综合分从 58.1679 降至 57.7848。问题不是召回完全失败，而是证据密度和成本结构失衡。

若 V12 官网准确率明显提升，V13 应：

1. 只对首轮与 V11 冲突且证据槽位不足的题执行审计。
2. 审计复用首轮 evidence context，避免重新检索和重复长 Prompt。
3. 将稳定的单选/判断题降为一次无 thinking 裁决。
4. 扩充人工开发集，按领域记录真实错误类型，而不是继续依赖模型置信度。
