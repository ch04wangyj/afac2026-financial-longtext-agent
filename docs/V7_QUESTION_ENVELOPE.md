## 问题

V6 已能判断多数选项事实，但“事实本身成立”不总等于“满足题干要求的集合”。
典型案例是 `reg_a_011`：记录保存至调查结束是法规真命题，却不属于题干要求的
内部审批或金额门槛。

## 实现

`agent/reasoning/question_envelope.py` 固定输出：

- `selection_rule`：选择正确项或错误项；
- `focus`：去除通用问句外壳后的题干关系；
- `scope_gate_enabled`：是否为显式集合归属题；
- 跨文档全称、比较和复合命题标记。

只有“哪些情形需要审批”“哪些产品可以赔付”等题启用双层协议：

1. `fact_truth`：选项完整事实是否被原文支持；
2. `applicable`：该事实是否满足题干要求的类别、关系、主体和口径；
3. Python 根据两层结果与选择方向确定入选状态。

普通“哪些说法正确”继续使用 V6 的单层 `truth`，避免无约束适用性判断。

## 实验

| 实验 | 结果 | 决策 |
|---|---:|---|
| 全 100 题启用双层协议 | 42 题相对 V6 漂移，`485,509` Token | 淘汰 |
| 仅 6 道集合归属题启用 | 6/6 复现 V6 答案 | 保留 |
| V6 分数反推 | `85/100`，显示分误差 `0.007` | 纳入工具 |
| `reg_a_003: B -> A` | 官网净增 1 题 | 纳入 V7 |

本轮结果与 SURE-RAG 的“检索不等于验证”一致，也吸收了
[S2G-RAG](https://aclanthology.org/2026.acl-long.1185/) 的结构化缺口思想；
本项目不引入额外模型，只把范围和缺口编译为确定性契约。
