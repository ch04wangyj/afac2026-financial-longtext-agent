# V12：条件标签覆盖复盘（已证伪）

## 官网结果

| 项目 | V11 | V12 |
|---|---:|---:|
| 正确题数 | 88 | 85 |
| Token | 327,052 | 327,052 |
| 官网综合分 | 86.2732 | 83.3320 |
| 答案变化数 | - | 5 |

V12 相对 V11 净损失 3 题。此前文档声称的 `89-93/100` 条件区间无效，不得再用于
候选选择或项目成果表述。

## 根因

1. `res_a_018=B` 由原文数值支持，但原文语义真值不等于比赛隐藏标签，不能作为
   MILP 固定条件。
2. V1/V2 本地快照无法与后续官网结果形成联合可行解，说明它们并非可确认的逐题
   官网提交副本，却被旧流程用于硬约束。
3. SciPy 1.11/HiGHS 在部分固定整数变量问题上让 `presolve` 返回伪可行解；复验时
   发现其违反多个官网正确题数等式 2 题。
4. 五个答案同时变化，只能观测净分差，无法把单题语义置信度升级为逐题官方标签。

## 修复

- `configs/leaderboard_runs.json` 记录快照可信状态、SHA-256、Token 和正确题数。
- 只有 `verified_submission` 可进入排行榜硬约束；V1-V3 已退出。
- MILP 快速解必须复验边界、整数性和全部原始线性约束；失败后关闭 presolve 重算。
- `source_semantic`、`model_consensus`、`leaderboard_forced` 和 `official_label`
  分层，前两类禁止生成隐藏标签固定条件。
- 发布候选必须报告全部可行正确题数和最坏结果，不再只报告乐观上界。

V12 配置与产物继续保留为负向实验，不能再次提交。
