## 一、赛题介绍

金融行业存在大量专业性强、篇幅长、结构复杂的文本（保险条款、监管法规、债券募集说明书、上市公司年报、行业研报等），直接全文输入模型会导致 Token 消耗巨大、推理成本难以控制；简单截断或摘要又容易丢失关键事实。

本赛题要求构建一个**金融长文本 Agent**，在不修改基座模型参数的前提下，设计 Agent 层面的**记忆流转、动态压缩和上下文优化策略**，在尽可能少的 Token 消耗下准确回答金融长文档相关的选择题和判断题。

**Baseline 参考**（基于 Qwen-plus）：

| 组别 | 正确/样本 | 准确率 | Token 消耗 |
|------|-----------|--------|------------|
| A 组 | 17/100 | 17.0% | 3,628,186 |
| B 组 | 13/100 | 13.0% | 3,884,045 |
| 总计 | 30/200 | 15.0% | 7,512,231 |


| 字段 | 类型 | 说明 |

### 2.4 文档形式

每条文档元数据包含 `doc_id`、`path`、`title` 等字段，对应原始 PDF 文件。长文本来源包括公开金融监管法规、公司合规指南、财务报告、保险条款与产品说明书、金融合同、法律文档和行业研究报告。文本刻意**保留较长篇幅和分散关键信息点**，考查 Agent 的压缩、检索和跨段推理能力。

## 三、答案格式要求

| 题型 | answer_format | 提交格式 | 示例 |
|------|---------------|----------|------|
| 单选题 | `mcq` | 单个大写字母 | `A` |
| 多选题 | `multi` | 多个大写字母，按字母顺序排列，**无分隔符** | `ABC` |
| 判断题 | `tf` | 单个大写字母 | `A` 或 `B` |

**评测规则**：

- 单选题和判断题以**首个有效答案字母**为准
- 多选题将答案字母**去重并排序**后与标准答案作**完全匹配**
- 多选题**不设部分分**：漏选、错选、多选均计为错误
- 答案为空、包含非法字符、顺序不规范 → 直接判错

## 四、提交格式

`answer.csv` 需同时包含每道题答案和 Token 消耗统计：

| qid | answer | prompt_tokens | completion_tokens | total_tokens |
|-----|--------|---------------|-------------------|--------------|
| summary | | 3627557 | 629 | 3628186 |
| ins_a_001 | B | 37201 | 1 | 37202 |
| ins_a_002 | A | 33063 | 1 | 33064 |
| ... | ... | ... | ... | ... |

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `qid` | String | 题目编号，第一行为 `summary` 汇总行 |
| `answer` | String | 答案字符串（summary 行留空） |
| `prompt_tokens` | Number | 全部作答过程消耗的输入 Token 总数 |
| `completion_tokens` | Number | 全部作答过程消耗的输出 Token 总数 |
| `total_tokens` | Number | prompt_tokens + completion_tokens |

> ⚠️ Token 统计必须覆盖回答全过程中的**所有模型 API 调用**（检索摘要、上下文压缩、证据判断、答案生成、自检等）。

## 五、评分公式

**准确率**：

$$\text{Accuracy} = \frac{\text{Correct}}{\text{Total}}$$

**Token 得分**（TokenBudget = 5,000,000）：

$$\text{TokenScore} = \max\left(0,\ \min\left(1,\ \frac{5{,}000{,}000 - \text{TotalTokens}}{5{,}000{,}000}\right)\right)$$

**最终得分**：

$$\text{FinalScore} = 100 \times \text{Accuracy} \times (0.7 + 0.3 \times \text{TokenScore})$$

> 准确率决定主体得分，Token 效率最多影响 **30%** 的加权系数。若 TotalTokens ≤ 0 或未提交有效统计，TokenScore 计为 0。

**排名优先级**：准确率 → TotalTokens（越低越好）→ 提交时间（越早越好）

## 六、推荐解题流程

```
文档预处理 → 文档索引 → 题目解析 → 证据检索 → 推理作答 → 答案校验 → 结果汇总
```

| 步骤 | 要点 |
|------|------|
| **1. 文档预处理** | 抽取 PDF 文本，保留标题、页码、章节、表格和条款编号 |
| **2. 文档索引** | 按领域构建关键词索引和结构化字段索引（⚠️禁止embedding） |
| **3. 题目解析** | 识别题型、关键词、实体、时间、金额、指标、候选文档 |
| **4. 证据检索** | 根据题目和选项分别检索证据，必要时多轮查询 |
| **5. 推理作答** | 让模型基于证据逐项判断选项真假 |
| **6. 答案校验** | 按题型约束输出合法答案字母，去重排序、格式检查 |
| **7. 结果汇总** | 生成符合提交格式的 answer.csv |

**重点优化方向**：

- 文档解析质量：提升 PDF 文本抽取、表格处理和版面结构还原
- 检索策略：构建面向金融术语、条款编号和指标名称的混合检索
- 证据聚合：支持同题多文档、多段落证据的联合判断
- 答案约束：对模型输出进行字母抽取、排序、去重和合法性检查
- 领域适配：针对保险、监管、合同、财报、研报分别设计提示词和后处理规则
